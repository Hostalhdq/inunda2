Protocolo de Prevención y Teléfonos Útiles

Acciones preventivas ante eventos hidrometeorológicos severos en Concepción

132 Bomberos Concepción 133 Carabineros 131 SAMU Salud Emergencias Mun. Concepción
Antes de las Lluvias
  • Limpiar canaletas y rejillas de evacuación en su propiedad.
  • Retirar hojas y desechos acumulados en los sumideros de su calle.
  • Verificar sellado de puertas frente a posibles anegamientos.
Durante la Lluvia Intensa
  • Evitar transitar a pie o en vehículo por el paso bajo nivel UBB.
  • Mover vehículos a zonas con mayor cota de elevación si sube el estero.
  • Sintonizar canales oficiales de SENAPRED y Municipalidad.
En caso de Desborde
  • Desconectar suministros de energía eléctrica a nivel de piso.
  • Evacuar hacia sectores altos (hacia sectores consolidados del Collao).
  • Notificar en la app la situación exacta para guiar socorristas.

Guía Técnica: Lo que falta para Puesta en Producción 100%

1 Conectar Base de Datos Supabase

Para que los reportes se compartan en tiempo real entre todos los visitantes (y no solo en el navegador de quien los crea), crea un proyecto gratis en supabase.com y corre esto en su SQL Editor: 

CREATE TABLE reportes_collao (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  latitud FLOAT NOT NULL,
  longitud FLOAT NOT NULL,
  categoria TEXT NOT NULL,
  nivel_severidad TEXT NOT NULL,
  descripcion TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  confirmaciones INT DEFAULT 1,
  resuelto BOOLEAN DEFAULT FALSE,
  resuelto_at TIMESTAMPTZ
);

-- El sitio no tiene login: cualquier visitante puede reportar,
-- confirmar y marcar como resuelto (mismo criterio que el resto
-- de la app). RLS abierto a todos, sin autenticación:
ALTER TABLE reportes_collao ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Lectura publica" ON reportes_collao FOR SELECT USING (true);
CREATE POLICY "Insercion publica" ON reportes_collao FOR INSERT WITH CHECK (true);
CREATE POLICY "Actualizacion publica" ON reportes_collao FOR UPDATE USING (true);

Luego ve a Project Settings → API, copia la "Project URL" y la "anon public" key, y pégalas en config.js (en la raíz del repo). La "anon" key está pensada para exponerse en el navegador — la protección real la dan las políticas de arriba, no el secreto de la key.

2 Automatizar el Dato de la DGA (Estero Nonguén)

La DGA sí tiene una estación en este tramo (código 08220008-8, "Estero Nonguen Frente U. Del Bio Bio", lecturas cada 30 min), pero su portal (snia.mop.gob.cl) exige resolver un captcha para ver los datos, así que no se puede leer automáticamente. Dos caminos para automatizarlo de verdad:

  • Pedir a la DGA acceso institucional/API sin captcha para esta estación (contactar por "Servicios de Información" en dga.mop.gob.cl).
  • Instalar un sensor propio (ultrasónico LoRaWAN) y conectarlo por webhook:
// Endpoint HTTP Webhook para Sensor de Nivel
POST /api/v1/telemetria/estero-nonguen
Headers: { "X-Sensor-Key": "SECURE_KEY" }
Payload: { "nivel_metros": 0.85, "bateria": 98 }

Mientras tanto, la app usa el nivel cargado a mano en la pestaña "Real vs Simulación" (alguien revisa snia.mop.gob.cl y mueve el slider al valor real).
